## Design Context

### Users
Analysts, data scientists, and engineers who upload datasets and interrogate them via natural language. They are technical enough to appreciate seeing the agent's code, SQL, and output in detail — and they enjoy when a tool has personality. They care about correctness and speed, and they want to *feel* the system working for them.

### Brand Personality
**Three words**: Authoritative. Precise. Delightful.

The product is a professional analytical chatbot for serious data work — but it has a sense of humor. It should feel like a premium financial data terminal that someone accidentally made fun. Every element must earn its place, but earning its place includes making the experience enjoyable to use, not just trustworthy.

### Emotional Goal
**Delight** — "That was fun and surprisingly fast." Users should finish an analysis feeling like the tool made them smarter *and* put a small smile on their face. Confidence is table stakes; delight is the differentiator.

### Aesthetic Direction
**Goldman Sachs terminal × modern dark SaaS × dry wit**

- Dense, data-forward layouts. Information density over decoration.
- Tabular numerals and monospace for all numeric values.
- Sharp typographic hierarchy: black-weight numbers, light labels.
- Color is semantic only — emerald for positive, rose for negative, amber for active/accent.
- Borders are functional (structural separators), not decorative.
- Radius: tight (4–8px) on all components. Never round.
- No ambient orbs, no glassmorphism, no gradient text, no synthwave neon.
- Motion is purposeful — progress panel is meant to be "enjoyable to watch."
- **Humor is a first-class feature**: rotating copy, fun verbs during tool execution, dry quips on completion. Keep it prominent and expand it.

**Progress panel philosophy**: Dev tool transparency. Show everything — live code, SQL, stdout output. Power users love seeing the agent work in detail. The code block while running should feel like watching the matrix.

**Anti-references**: Generic SaaS dashboard templates (Notion, Figma), neon/synthwave glow cards, purple-to-blue gradient cards, minimal/sparse whitespace-heavy designs, apps that take themselves too seriously.

**References**: Bloomberg Terminal, Goldman Sachs internal tools, Linear (precision + hierarchy), Stripe Dashboard (data confidence), early Vercel dashboard (technical elegance with personality).

### Design Principles
1. **Data is the hero** — Metrics, values, and trends are always the biggest, boldest element. Labels, icons, and chrome stay subordinate.
2. **Color as signal, not decoration** — Only use color to convey meaning (positive/negative/warning/active). Never for visual interest alone.
3. **Density earns trust** — Compact, information-rich layouts signal expertise. Oversized cards with lots of padding signal consumer tools.
4. **Monospace for precision** — All numeric values use tabular-nums + monospace accent. Numbers should feel machine-precise, not typeset.
5. **Hierarchy through weight, not effects** — Use font-weight contrast (900 vs 400) rather than shadows, glows, or gradients to create hierarchy.
6. **Humor is intentional, not accidental** — Rotating copy, fun verbs, dry completion quips are a deliberate product choice. They should surprise and delight, never feel like filler.
7. **Transparency builds trust** — Show the work. Live code, SQL previews, stdout output, step-by-step progress. Users who can see the agent thinking trust the output more.
